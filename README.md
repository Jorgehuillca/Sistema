# 📌 Proyecto: Aplicación Web - Control de Versiones con Git

## 📖 Descripción

Este proyecto consiste en la implementación de una aplicación web básica, gestionada mediante el sistema de control de versiones Git y publicada en GitHub.

El propósito es aplicar los conceptos fundamentales de control de versiones, como commits, ramas y repositorios remotos, dentro de un flujo de trabajo real.

---

## 🎯 Objetivo

Aplicar el flujo de trabajo esencial de Git y GitHub para gestionar versiones de software de manera eficiente y colaborativa.

---

## 🛠️ Tecnologías utilizadas

* HTML
* Git
* GitHub
* Visual Studio Code

---

## 📂 Estructura del proyecto

* `index.html`: Página principal de la aplicación
* Otros archivos: Recursos adicionales del proyecto web

---

## 🚀 Uso del proyecto

1. Clonar el repositorio
2. Abrir el archivo `index.html` en el navegador

---

## 🔀 Flujo de trabajo con Git

Durante el desarrollo se aplicaron los siguientes conceptos:

* Inicialización del repositorio (`git init`)
* Registro de cambios (`git add`, `git commit`)
* Uso de ramas:

  * main
  * desarrollo
  * pruebas
  * documentacion
* Sincronización con repositorio remoto (`git push`, `git pull`)

---

## 📝 Buenas prácticas aplicadas

* Uso de mensajes de commit descriptivos:

  * feat: nuevas funcionalidades
  * fix: corrección de errores
  * docs: documentación
* Organización del código
* Uso de ramas para trabajo paralelo
